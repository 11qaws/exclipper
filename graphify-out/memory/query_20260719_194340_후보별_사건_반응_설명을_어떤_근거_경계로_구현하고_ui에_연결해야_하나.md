# Q: 후보별 사건·반응 설명을 어떤 근거 경계로 구현하고 UI에 연결해야 하나?

## Answer

App의 후보 카드가 fast audio·chat·visual과 세션 전용 Pass B/audio-event를 candidate ID 및 proposal binding으로 검증한 뒤, 결정적 CandidateEvidenceExplanation으로 투영한다. 전사·혼합 오디오는 사건·주체·인과로 승격하지 않고 replay cue로만 사용하며, malformed overlay는 카드별 fast evidence로 fallback한다. 후보 1개부터 정밀 기능을 보이고 2개 이상에서만 ranking을 노출한다.
