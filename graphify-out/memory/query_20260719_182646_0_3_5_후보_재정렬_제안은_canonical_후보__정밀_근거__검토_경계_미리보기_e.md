# Q: 0.3.5 후보 재정렬 제안은 canonical 후보, 정밀 근거, 검토·경계·미리보기·export를 어떻게 안전하게 분리해야 하는가?

## Answer

canonical candidates는 변경하지 않고 CandidateRankingProposal의 완전한 ID permutation을 별도 CandidateRankingViewState에 저장한다. 제안 도착은 화면을 바꾸지 않으며 명시적 APPLY/UNDO만 active projection을 바꾼다. transcript는 랭킹 0점, 부분 AST run은 전체 0점이며 review/boundary/preview는 candidate ID로 보존하고 export는 시작 시각순을 유지한다.
