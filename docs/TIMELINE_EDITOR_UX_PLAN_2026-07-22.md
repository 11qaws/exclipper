# ExClipper 동적 타임라인·후보 검토 워크스페이스 계획

- 작성일: 2026-07-22
- 목표 버전: 0.3.41
- 대상 화면: 최대화된 데스크톱 브라우저의 분석 결과·후보 검토 화면
- 제품 관점: 몇 시간짜리 방송을 직접 훑지 않고 AI가 종합한 짧은 후보만 결정하는 개인 편집 어시스턴트

## 1. 현재 화면에서 확인된 문제

### 1.1 후보 검토 영역의 비대칭

- 좌측 미리보기와 우측 후보 정보가 서로 다른 레이아웃 컨테이너에 속한다. 우측에만 이전/다음 내비게이션 한 줄이 먼저 있어 두 카드의 시작선이 맞지 않는다.
- 좌측은 영상 비율과 짧은 안내에 의해 높이가 일찍 끝나고, 우측은 사건·반응 근거를 펼칠 때 페이지 아래로 계속 늘어난다.
- 우측 증거를 펼치는 행동이 전체 2열 레이아웃의 높이를 바꾸므로 좌우 관계가 더 약해진다. 사용자는 같은 후보의 ‘화면’과 ‘판단 근거’를 비교하는 대신 서로 다른 길이의 문서를 보는 느낌을 받는다.
- 해결 목표는 내용을 잘라 같은 높이로 만드는 것이 아니다. 두 패널의 **외곽 높이와 시작·끝 기준선**을 같게 하고, 긴 근거만 우측 패널 내부에서 스크롤되게 한다.

### 1.2 타임라인의 시각 부호와 데이터가 분리됨

- 주제 띠의 색이 배열 순번 `index % 4`에서 왔기 때문에 색에 의미가 없다. 같은 종류의 사건도 순서가 달라지면 다른 색이 되고, 범례로 설명할 수 없다.
- 주제에는 `titleKo`, `summaryKo`, `kind`, `salience`, `uncertaintiesKo`, 관련 후보 ID가 있지만 현재 띠에는 제목 일부와 hover tooltip만 남고 대부분의 분석 결과가 사라진다.
- 의미 단서도 번호·색만 먼저 보이고 `eventSummaryKo`, `whyThisMomentKo`, `evidenceCueKo`, `confidence`, 불확실성은 떨어진 상세 목록을 열어야 볼 수 있다.
- 기존 파란 선은 처음에는 방송 전체 시간축을 상징했지만, 30분 눈금과 주제 범위 트랙이 생긴 뒤 별도 정보가 없는 장식선이 되었다. 큰 후보 원을 받치는 역할만 남아 초기 후보를 과도하게 강조한다.

### 1.3 분석 중 상태와 최종 편집 상태가 섞임

- fast pass 후보가 맥락 판단 전에 큰 원으로 표시되어 사용자가 이를 최종 추천으로 오해하기 쉽다.
- 맥락 탐색의 실제 진행 위치와 주제가 형성되는 과정은 보이지 않으면서, 아직 임시인 후보만 먼저 보인다.
- 최종 후보를 공개하기 전에는 초기 점수를 ‘잠재 신호’로만 유지하고, 분산 탐색 셀과 확인된 주제 범위를 중심으로 보여야 한다.

## 3. 설계 원칙

1. **시간은 하나의 x축만 가진다.** 파란 장식선을 제거하고 상단 time ruler와 세로 grid가 모든 트랙의 위치 기준이 된다.
2. **트랙 하나에는 한 종류의 질문만 둔다.** 점수는 ‘어디가 수상한가’, 탐색 셀은 ‘어디까지 읽었나’, 주제는 ‘무슨 내용이 이어졌나’, 의미 단서는 ‘어디를 더 볼 가치가 있나’, 후보는 ‘편집자가 결정할 장면은 무엇인가’를 답한다.
3. **색에는 고정된 의미가 있어야 한다.** 순번 기반 색을 금지하고 semantic family에만 색을 쓴다. 색 옆에는 항상 텍스트 라벨을 둔다.
4. **범위를 선택하면 데이터가 남아 있어야 한다.** 주제·단서 클릭은 source range, 분석 요약, 근거, 확신, 불확실성을 inspector에 공개한다.
5. **초기 신호와 최종 후보를 다른 계층으로 본다.** 맥락 종합 전에는 후보 marker를 렌더하지 않는다. 종합 뒤에도 `deprioritized`는 작고 낮은 대비로 보존한다.
6. **같은 후보의 양쪽 패널은 같은 외곽 크기다.** 공용 내비게이션 아래에 1:1 열을 두고 동일한 고정 높이를 사용한다. 긴 내용은 내부 스크롤로 해결한다.
7. **동적 표현은 실제 상태만 반영한다.** ASR partial/gap은 즉시 표시하지만, batch로 받은 Semantic Chapter를 모델 streaming이라고 표현하지 않는다.

## 4. 새 정보 구조

### 4.1 단일 실시간 분석 패널

- 큰 영역에는 현재 단계 번호, 현재 작업 제목, 한 줄 상태, 하나의 progress bar만 둔다.
- `빠른 탐색 / 전체 맥락 / 후보 종합 / 편집자 선택`은 작은 네 점 rail로만 남긴다.
- 분석 중 초기 후보 개수, 검토 통계, 고급 후보 도구는 숨긴다.

### 4.2 방송 전체 지도

위에서 아래로 다음 고정 순서를 사용한다.

1. **Time ruler**: 00:00, 30분 간격, 영상 끝. 세로 grid를 모든 트랙에 연결한다.
2. **잠재 신호 overview**: 낮은 높이의 흐릿한 막대. clip 확률이 아니라 fast-pass 상대 강도임을 명시한다.
3. **맥락 탐색 cells**: `queued | active | complete | gap`. active cell은 pulse, gap은 빗금이다.
4. **주제 chapters**: 실제 start/end range를 채우는 큰 segment. 제목과 semantic family 라벨을 함께 표시하고 클릭 가능하게 한다.
5. **의미 단서 leads**: 작은 range marker. category 번호 대신 짧은 유형 라벨 또는 아이콘+번호를 사용한다.
6. **최종 후보 markers**: publication gate 뒤에만 표시한다. 시간순 번호를 유지하되 추천, 검토 필요, 낮은 우선순위를 크기와 테두리로 구분한다.

별도 파란 실선은 제거한다. 현재 선택 위치는 모든 트랙을 가로지르는 1px playhead와 상단 timecode bubble로 표현한다.

### 4.3 의미 색 체계

주제의 세부 kind를 네 가족으로 묶는다.

| 색 가족 | 포함 kind | 표시 라벨 |
|---|---|---|
| 파랑 | `main-event`, `reaction` | 주요 사건·반응 |
| 초록 | `quiet-achievement`, `setup-and-payoff` | 성취·회수 |
| 보라 | `story-progress`, `context-shift`, `running-gag` | 흐름·전환 |
| 회색 | `other` | 일반 맥락 |

의미 단서는 기존 category별 색을 쓰되 범례에 `특징적 반응 / 조용한 성취 / 설정과 회수 / 반복 개그 / 맥락형 사건 / 사과·해명`을 모두 표시한다. 좁은 화면이나 고대비 모드에서는 색 외에 border pattern과 텍스트를 함께 사용한다.

### 4.4 선택 항목 inspector

타임라인 아래 한 줄 높이의 기본 상태와 확장 가능한 정보 카드로 구성한다.

- 주제 선택: 제목, 시간 범위, semantic family, 중요도, `summaryKo`, 관련 후보 수, `uncertaintiesKo`.
- 의미 단서 선택: category, 시간 범위, `eventSummaryKo`, `whyThisMomentKo`, `evidenceCueKo`, confidence, `uncertaintiesKo`.
- 후보 선택: 기존 후보 카드와 재생 편집기로 이동하고 공통 playhead를 candidate peak에 맞춘다.
- inspector의 ‘이 구간 보기’는 원본이 연결됐을 때 해당 start time으로 재생한다. 연결되지 않았으면 버튼을 비활성화하고 분석 텍스트는 그대로 보여 준다.

## 5. 후보 검토 워크스페이스

### 5.1 데스크톱 구조

```text
┌──────────────────────────────────────────────────────────────────┐
│ 이전 후보                  후보 1 / 17                  다음 후보 │
├───────────────────────────────┬──────────────────────────────────┤
│ 선택 후보 미리보기            │ 후보 사건·반응·결정                │
│                               │                                  │
│ 16:9 video / 연결 안내        │ 고정 header + 결정 버튼            │
│                               │                                  │
│ 재생·현재 구간 안내            │ 내부 스크롤 근거·경계·AI 상세       │
└───────────────────────────────┴──────────────────────────────────┘
```

- 공용 내비게이션은 두 열 위 전체 폭을 사용한다. 우측만 한 줄 아래에서 시작하는 문제를 없앤다.
- 두 열은 정확히 `1fr 1fr`로 두고 동일한 높이 `clamp(620px, 76vh, 820px)`를 사용한다.
- 좌측과 우측은 `min-height: 0`을 가진 grid item이다. 우측 확장 내용은 `.rh-candidate-column` 내부에서만 스크롤된다.
- 좌측 영상/연결 안내 영역은 남는 높이를 채운다. 원본이 없을 때도 우측과 동일한 하단 기준선을 유지한다.
- 우측 후보 header와 승인·제외 action은 panel 상단에 고정하고, 근거·경계·AI 상세만 그 아래에서 스크롤한다. 현재 페이지 전체에 붙어 있는 sticky action bar는 패널 내부 sticky로 한정한다.
- 최대화 화면에서 두 패널을 동시에 비교하는 것이 기본이다. 1120px 아래에서만 단일 열로 접고 고정 높이를 해제한다.

### 5.2 높이 규칙

- ‘내용이 짧은 쪽을 억지로 늘이는 빈 카드’가 아니라 같은 작업 surface의 frame을 맞춘다.
- 내부 콘텐츠가 frame보다 짧으면 자연스러운 여백을 허용하고, 길면 해당 pane 내부 스크롤을 사용한다.
- details를 열고 닫아도 outer workspace 높이, 타임라인 위치, 왼쪽 video frame은 움직이지 않는다.
- 후보 전환 시 내비게이션은 고정되고 우측 내부 스크롤만 맨 위로 초기화한다.

## 6. 동적 맥락 탐색 로직

- chronological chunk 집합은 그대로 유지하되 실행은 황금비 seed + farthest-point 순서로 분산한다.
- 한 셀에서 사건/인과/감정 신호가 충분하면 미처리 인접 셀 최대 2개를 앞으로 당긴다. 두 인접 셀 사이에 전역 probe 하나를 유지한다.
- 음악 표기나 일반 진행 문장만으로는 이웃 확장을 시작하지 않는다.
- partial은 source fence와 함께 즉시 checkpoint하고 UI cell을 complete로 바꾼다. gap은 별도 상태로 남긴다.
- 최종 transcript와 context request는 다시 source-time 순서로 합친다. 실행 순서 때문에 이야기 순서가 뒤섞이지 않는다.
- 이 변경은 같은 planned cells의 순서만 바꾸므로 API 호출 수와 duration 비용을 늘리지 않는다.

## 7. 자체 검토 결과

### 수용

- 편집 도구의 time ruler, grouped tracks, marker detail panel, overview+detail 동기화를 수용한다.
- 색을 semantic family에만 고정하고 모든 구간의 실제 분석 텍스트를 선택 inspector로 연결한다.
- 좌우 높이는 fixed outer frame + internal scroll로 맞춘다. 긴 근거를 잘라서 맞추지 않는다.
- 초기 후보 marker는 publication gate 뒤로 미룬다.

### 수정 수용

- 자유 zoom은 긴 방송에 유용하지만 초심자에게 현재 위치를 잃게 할 수 있다. 1차 구현은 전체 12시간 fit + 30분 ruler를 유지하고, 이후 선택 범위 확대와 ‘전체 보기’ 복귀 버튼을 한 쌍으로 도입한다.
- 모든 사건 category에 독립 색을 주면 범례가 다시 복잡해진다. chapter는 4개 색 가족으로 줄이고, lead만 6개 category를 사용한다.
- topic reveal은 동적으로 보이지만 실제 context API는 batch 응답이다. ‘AI가 지금 이 주제를 발견했다’가 아니라 ‘찾은 주제 지도를 펼치는 중’으로 표현한다.

### 거부

- 파란 선을 유지한 채 트랙을 더 쌓는 안은 정보 중복이라 거부한다.
- 색만으로 종류를 설명하는 안, hover tooltip에만 분석 결과를 숨기는 안, 우측 details가 페이지 전체 높이를 늘리는 안을 거부한다.
- 12시간 전체의 연속 썸네일 filmstrip은 브라우저 디코더·메모리 비용과 시각 혼잡 때문에 거부한다. 최종 후보와 선택한 주제에만 대표 화면을 준비한다.

## 8. 구현 순서

1. 현재 단계 패널과 candidate publication gate를 완성한다.
2. ASR distributed order, adaptive neighbor queue, live cell state를 단위 테스트한다.
3. timeline의 파란 실선을 제거하고 ruler/track 좌표계를 재배치한다.
4. chapter kind → semantic family mapping을 순수 함수로 만들고 테스트한다.
5. topic/lead 선택 state와 inspector를 연결한다.
6. 후보 내비게이션을 공용 상단 row로 이동하고 1:1 동일 높이 pane을 적용한다.
7. 저장 음식 토크 세션으로 17개 후보, 7개 주제, 28개 의미 단서를 복원해 최대화 화면을 검증한다.
8. details 확장 전후 outer workspace 높이, 좌우 top/bottom baseline, 내부 scroll을 브라우저에서 측정한다.
9. reduced motion, keyboard focus, forced colors, 1120px/760px responsive를 확인한다.

## 9. 추가 검토: 긴 방송 요약과 주 진행 스트리머 프로필

- 수십만 자의 근거를 300~500자로 압축하던 계약은 편집자가 전체 맥락을 판단하기에 지나치게 짧다. 새 분석은 방송 흐름 서술을 **600~1,000자**로 늘리고, 시간 순서·주요 주제 변화·반복 소재·클립 판단에 영향을 준 사건을 남긴다.
- 방송 요약과 별도로 `주 진행 스트리머 프로필`을 구조화한다. 방송 안에서 관찰된 진행 역할, 말투와 상호작용 방식, 반복 관심사, 반응 패턴, 채팅·게스트와의 관계를 300~500자로 정리하고 근거 단서와 불확실성을 함께 저장한다.
- 정확한 이름은 제공된 출연자 명단이나 방송 내용에서 명시적으로 확인될 때만 사용한다. 확인되지 않으면 `주 진행 스트리머`라고 쓰며, 나이·성별·국적·건강 등 민감하거나 근거 없는 신상은 추정하지 않는다.
- 화면은 `방송 전체 서술`과 `AI가 파악한 주 진행 스트리머`를 같은 높이의 1:1 정보 카드로 배치한다. 각 카드 내부가 길어질 때만 스크롤하거나 펼쳐 보며, 바깥 레이아웃의 기준선은 유지한다.
- 이전의 짧은 응답이 새 분석으로 재사용되지 않도록 맥락 스키마와 라우팅 캐시 버전을 올린다. 오래된 세션은 프로필을 꾸며내지 않고 재분석 필요 상태를 명확히 표시한다.

## 10. 추가 검토: 동일 높이 후보 미니 카드

- 후보 목록은 후보마다 콘텐츠 양에 따라 높이가 달라지는 masonry 형태를 쓰지 않는다. 데스크톱에서 모든 미니 카드는 하나의 대표 높이와 동일한 내부 행 구조를 사용한다.
- 썸네일, 후보 번호·상태, AI 출처, 한 줄 사건 설명의 영역을 고정한다. 제목은 최대 1줄, 출처는 최대 2줄, 설명은 최대 2줄로 제한하고 넘치는 텍스트는 말줄임한다.
- 카드의 짧은 문구는 `어떤 사건/반응인지`를 먼저 보여 주도록 정규화한다. 원문 전체와 판단 근거는 카드에서 삭제하지 않고 선택 inspector와 후보 상세 패널에 그대로 보존한다.
- 미리보기 준비 상태도 같은 썸네일 프레임을 사용한다. 후보 17개처럼 마지막 행이 덜 차더라도 각 카드의 높이·썸네일 비율·텍스트 기준선은 앞 행과 동일해야 한다.
- 키보드 포커스와 선택 테두리가 카드 크기를 바꾸지 않아야 하며, 작은 화면에서는 폭만 한 열로 바뀌고 카드 높이 계약은 유지한다.

### 추가 완료 기준

- 새 분석 결과의 전체 방송 서술이 600~1,000자 목표를 사용하고, 근거 기반 주 진행 스트리머 프로필·근거·불확실성이 저장 후 새로고침에도 보존된다.
- 후보 미니 카드 1~17의 측정 높이가 동일하며, 긴 한국어 문구를 주입해도 어느 카드도 한 줄 이상 늘어나지 않는다.

## 11. 완료 기준

- 분석 중 `.rh-timeline-marker`, 후보 카드, 후보 편집기가 0개이고 잠재 점수·탐색 셀·주제만 보인다.
- 맥락 종합 뒤 후보 marker와 카드가 한 번에 나타나며 canonical ID·review state·boundary가 바뀌지 않는다.
- chapter 색은 배열 순번이 아니라 kind family로 결정된다. 범례와 inspector가 같은 라벨을 사용한다.
- 주제 7개와 의미 단서 28개 각각을 클릭해 시간 범위와 저장된 분석 문장을 확인할 수 있다.
- 별도 파란 장식선이 없고 time ruler/grid가 모든 트랙의 위치 기준이다.
- 1440px 이상에서 좌우 pane의 top/bottom 좌표 차이가 1px 이하이고, 우측 details 확장 전후 outer workspace 높이가 변하지 않는다.
- 전체 strict TypeScript, ESLint warning 0, 단위 회귀, production build, 공개 GitHub Pages browser smoke가 통과한다.
